# Gate Rules

## G0 Bootstrap

Pass condition: control files, source-of-truth documents, and the initial Next.js skeleton exist.

## G1 App skeleton

Pass condition: the app skeleton is stable enough for the first product-facing page and development checks. To be specified in a later ticket.

## G2 Domain model

Pass condition: core domain types exist without adding database, authentication, or HMRC API implementation. To be specified in a later ticket.

## G3 Spreadsheet bridge core

Pass condition: spreadsheet bridge rules are represented without arbitrary messy spreadsheet mapping. To be specified in a later ticket.

## G4 HMRC sandbox client

Pass condition: sandbox client work is implemented without production claims. To be specified in a later ticket.

## G5 Quarterly update payloads

Pass condition: quarterly update payload work is implemented for the approved scope only. To be specified in a later ticket.

## G6 Sandbox evidence readiness

Pass condition: evidence rules are met using actual HMRC sandbox API calls only. To be specified in a later ticket.
