# QL-006

Status: DRAFT

## Goal

Auth, tenant, audit, secrets, and HMRC access architecture.

Decide the minimum architecture needed before real HMRC sandbox work so production access is not blocked later by weak security, token handling, audit, or fraud-prevention design.

## Scope

- Decide auth approach for the sandbox-readiness MVP.
- Decide tenant boundary for the first individual-user path.
- Define audit events needed for HMRC connection, spreadsheet import, evidence review, and quarterly update sending.
- Define server-side secret and HMRC token handling requirements.
- Define fraud prevention header architecture.
- Define which parts must be implemented before sandbox testing and which can wait until production hardening.
- Write a run report.

## Out of Scope

- HMRC OAuth.
- HMRC API calls.
- Sandbox evidence.
- Full authentication implementation unless explicitly approved.
- Full database implementation unless explicitly approved.
- Spreadsheet parsing.
- Quarterly update payload submission.
- Final declaration or tax return features.

## Approval

Do not implement this ticket.
