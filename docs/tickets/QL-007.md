## Required technical requirements

### 1. Server-only HMRC config

Implement a server-only configuration boundary for HMRC sandbox settings.

It must:

- reject missing required sandbox config;
- reject malformed URLs;
- keep secrets server-side only;
- avoid exposing secrets in thrown errors;
- separate sandbox and production configuration;
- default to sandbox/local-safe behaviour;
- fail closed if required settings are missing.

### 2. Required environment variables

Define the minimum environment variables needed for QL-007 and future QL-008 sandbox calls.

Include placeholders only. Do not include real secrets.

The docs must explain:

- what each variable is for;
- whether it is required now or later;
- where it should be configured locally;
- where it should be configured in deployment;
- which values must never be committed.

### 3. HMRC API client scaffolding

Add a server-side HMRC client scaffold that can construct requests but does not perform unauthorised production calls.

It must:

- use the accepted sandbox base URL;
- support passing bearer tokens only as server-side values;
- support adding fraud-prevention headers;
- return structured request metadata suitable for tests;
- include safe error classes or result types where practical;
- avoid logging sensitive tokens, secrets, or raw fraud metadata.

### 4. Fraud-prevention-header assembly

Implement helper structure for fraud-prevention headers according to the accepted QL-006 architecture.

It must:

- separate client-collected values from server-derived values;
- make missing required values explicit;
- support safe redaction;
- avoid trusting spoofable client values as authoritative where server derivation is required;
- avoid writing raw sensitive fraud metadata to logs;
- include tests for required/missing/redacted behaviour.

### 5. OAuth scaffolding boundary

QL-007 may define OAuth URL construction only if needed for the foundation.

It must not:

- store tokens;
- create a full session/login system;
- persist user authorisation;
- claim HMRC connection is working end-to-end unless it actually is and was explicitly approved.

### 6. Local/demo vs HMRC sandbox evidence separation

Add or update docs/code comments so it is clear that:

- current UI evidence previews are local/demo placeholders;
- HMRC sandbox evidence must only be recorded from actual sandbox responses;
- local/demo evidence must not be mixed with sandbox evidence;
- production evidence must not be created in this ticket.

### 7. Browser exposure boundary

Ensure nothing added by QL-007 exposes these to browser/client code:

- HMRC client secret;
- HMRC access token;
- HMRC refresh token;
- encryption keys;
- session signing secrets;
- raw secret environment variables;
- unredacted fraud-prevention metadata;
- unrelated tenant/taxpayer data.
