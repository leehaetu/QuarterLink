# QL-001

Status: COMMITTED

## Goal

Stabilise the bootstrap and prepare the first product-facing web app skeleton so QuarterLink can move from repository setup into controlled web-app development.

This ticket is about the application foundation only. It must make the repo easier to build on without implementing HMRC, authentication, database, spreadsheet parsing, submission, billing, or production admin features.

## Product Context

QuarterLink is bridging-only software for Making Tax Digital for Income Tax. The web app should focus first on users who keep spreadsheet records and will later need to send quarterly updates.

The first app experience should feel like a product workspace, not a sales or marketing website. Public sales and marketing pages are outside this ticket.

## Scope

- Remove default create-next-app boilerplate from `README.md`.
- Correct documentation drift that refers to `app/page.tsx` instead of `src/app/page.tsx`.
- Keep the current Next.js App Router, `src` directory, TypeScript, Tailwind CSS, ESLint, npm, and Turbopack setup.
- Review the existing homepage and convert it, if needed, into a simple app-oriented starting screen rather than a sales page.
- Keep user-facing wording aligned with the HMRC wording guardrails.
- Preserve the placeholder architecture under `src/core` and `src/tax-regimes`.
- Add only lightweight UI scaffolding needed for future web-app work, such as static layout sections, empty states, and non-interactive navigation placeholders.
- Make any mock or preview information obviously local/static and not HMRC sandbox evidence.
- Keep all imported or displayed monetary figures out of scope.
- Run checks and write a run report.
