## Implementation Decisions

- QL-007 config accepts only sandbox HMRC origins:
  - `https://test-api.service.hmrc.gov.uk`
  - `https://test-www.tax.service.gov.uk`
- Production HMRC configuration is blocked in the QL-007 config helper.
- The request client only prepares request objects and safe metadata. It does not call `fetch`.
- OAuth scaffolding builds only an authorisation URL. It does not create routes, callbacks, sessions, token exchange, PKCE storage, or persistent authorisation.
- Safe metadata redacts bearer tokens, raw fraud-prevention headers, and query parameter values.
- Fraud-prevention-header assembly separates client-collected values from server-derived values.
- Missing `Gov-Client-Multi-Factor` and `Gov-Vendor-License-IDs` block assembly unless a later human/HMRC missing-data decision changes that rule.
- `.env.example` is tracked with placeholders only; `.env.local` and real `.env*` files remain ignored.
- `tsx` was added as the smallest local test runner so TypeScript tests can run through Node's built-in test framework.
